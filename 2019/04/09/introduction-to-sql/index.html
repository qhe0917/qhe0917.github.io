But after I opened the wiki page, there are so many new terms in the description, each needs a further explanation for me to understand the content. So, I plan to study to SQL first and answer those questions later in this blog.

RSQLite, use SQL in Rstudio

RSQLite is a package that contains SQL in it. The advange of using this package is we can work with data effciently dplyr. The dplyr works with the index to SQL db in R and extract or calculate the data after specifying the index in R. It is efficient to extract subset with complex restrictions.

However, R is not capable of handling very big dataset. One disadvantage of using RSQLite instead of SQL directly is possiblity of overflowing. We may extract a subset without knowing its size. It is possible that the subset extracted is too large to be handled by R. In this case, we may need to use other methods to handle the data.

It is not a best idea to create SQL database with R, since R is not capable of handling big database. The reason why we need to do that is sometimes we need to chop a big database into small chunk with R and save it as SQL, or attach multiple small db into a big sql.

So, when we know that the subset of SQL db we are interested in can be handled by R, RSQLite is a very efficient way to directly visit SQL and do the statistical analysis from it.
